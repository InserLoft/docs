**Is Inserloft only about building models?**
No. Inserloft builds a full ecosystem — models are one part, alongside the [Pixel Platform](/docs/platform/pixel), [APIs](/docs/apis/authentication), [Cleo](/docs/cleo/introduction), and [Research](/docs/research/research-areas). See [Philosophy](/docs/getting-started/philosophy).

**What's the difference between Kyro and Verty?**
[Kyro](/docs/models/kyro) is Inserloft's frontier family, optimized for capability. [Verty](/docs/models/verty) is a [Baby LLM](/docs/models/baby-llms), optimized for efficiency and open, local use.

**Why is a Project not called a repository?**
Because a Pixel [Project](/docs/platform/projects) is meant to hold more than code — models, datasets, documentation, and discussions all live in the same place.

**What license is Verty released under?**
The Apache License 2.0, chosen because it permits commercial use and drives broader adoption.

**Where can I find API pricing and limits?**
See [Rate Limits](/docs/apis/rate-limits) for request limits; pricing is managed per Organization on [Pixel](/docs/platform/pixel).
