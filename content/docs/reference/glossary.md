---

**Project** — the core organizational unit on Pixel, equivalent to a repository. See [Projects](/docs/platform/projects).

**Pixel** — Inserloft's main platform, combining elements of GitHub, Hugging Face, and an AI workspace. See [Pixel](/docs/platform/pixel).

**Cleo** — Inserloft's AI assistant. See [Cleo](/docs/cleo/introduction).

**Kyro** — Inserloft's frontier model family. See [Kyro](/docs/models/kyro).

**Verty** — Inserloft's first open-source model, a Baby LLM published by OpenGit. See [Verty](/docs/models/verty).

**Baby LLM** — a small, efficient, locally runnable model. See [Baby LLMs](/docs/models/baby-llms).

**OpenGit** — Inserloft's open-source organization. See [OpenGit](/docs/opengit/introduction).

**Model Card** — standardized documentation attached to a model. See [Model Cards](/docs/models/model-cards).

**Deployment** — a running, callable instance of a model or Project. See [Deployments](/docs/platform/deployments).
