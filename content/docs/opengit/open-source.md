Inserloft treats open source as a strategy, not an afterthought.

The clearest example is licensing: Verty moved to the **Apache License 2.0** specifically because it permits commercial use, which drives broader adoption than more restrictive research-only licenses. See [Licenses](/docs/company/security) and the [Verty](/docs/models/verty) page for more.

Open-source releases from OpenGit fall into a few categories:

- **Models** — open-weight releases like Verty
- **Datasets** — data shared alongside model releases for reproducibility
- **Tools and SDKs** — everything developers need to actually use what's released
