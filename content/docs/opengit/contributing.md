OpenGit welcomes external contributions across models, tools, and datasets.

The general workflow:

1. Find or open an Issue on the relevant OpenGit Project describing what you want to work on.
2. Fork or branch the Project on Pixel.
3. Submit your changes for review through the Project's standard review flow.
4. Once merged, changes are included in the next tagged Release.

For model contributions specifically — such as fine-tunes of [Verty](/docs/models/verty) — please include an updated [Model Card](/docs/models/model-cards) describing what changed and why.
